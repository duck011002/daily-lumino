# Lumino 安全 / 性能 / UI 评审报告

> 评审对象：`lumino_实施文档_v1.1_Codex版.md`（实施文档 v1.1）+ `backend/` 现有骨架
> 评审日期：2026-06-05
> 重要前提：当前仓库**尚无业务代码**，只有实施文档、`backend/pyproject.toml`、`backend/.env.example` 和一个空 venv。
> 因此本报告评审的是**设计层面**的问题。在代码生成之前修正这些点，成本最低、收益最大。

## 阅读指引

- 严重级别：**P0 致命** → **P1 高** → **P2 中** → **P3 低**
- 每条包含：现状 / 风险 / 解决方案（含可直接落地的代码或配置）
- 安全部分按严重程度排序；性能与 UI 部分分列在后半部分。

## 一、安全问题（按严重程度排序）

| 编号 | 级别 | 问题 | 一句话风险 |
|---|---|---|---|
| S1 | P0 | 真实数据库密码 / 凭据明文写进文档并入库 git | 任何拿到仓库的人可直连生产库 |
| S2 | P0 | 全站明文 HTTP，无 TLS | JWT、密码、聊天内容可被中间人窃取 |
| S3 | P0 | Cookie 鉴权但缺 CSRF 防护 | 可被跨站伪造请求 |
| S4 | P1 | JWT 无法吊销，logout 只清浏览器 cookie | token 泄露后 7 天内无法封禁 |
| S5 | P1 | 敏感配置"加密存储"未定义密钥来源与算法 | 易被实现成可逆弱加密 |
| S6 | P1 | 公开注册默认开放、邀请码可选 | 私密应用被陌生人注册 |
| S7 | P1 | 图片上传仅校验扩展名 / MIME | 可上传伪装的恶意文件 |
| S8 | P1 | 登录 / 注册 / 聊天接口无限流 | 暴力破解 + LLM 费用被刷爆 |
| S9 | P2 | 存储配额单行 read-modify-write 竞态 | 并发上传可超额 |
| S10 | P2 | 密码策略弱（8 位）、无失败锁定 | 易爆破 |
| S11 | P2 | JWT 解码未强制算法白名单 | 算法混淆 / alg=none 风险 |
| S12 | P2 | 缺少安全响应头（CSP / HSTS 等） | XSS、点击劫持面扩大 |
| S13 | P3 | `python-jose` 等依赖有已知维护 / CVE 问题 | 供应链风险 |
| S14 | P3 | 无审计日志、错误信息可能泄露内部地址 | 难追溯、信息泄露 |
